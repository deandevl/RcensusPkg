Introduction to RcensusPkg

Rick Dean

Abstract
Accessing the US Census Bureau’s database of survey data and map geometries via their API can be a challenge. The R package RcensusPkg can help by providing specific functions for checking on the availability of geographies, variables, datasets, and dates. In addition, the package provides easy access to the Bureau’s extensive Tiger Line map geometries and for joining the maps with Census Bureau survey data. This paper presents a basic step-by-step demonstration of how RcensusPkg could be applied to investigate computer and broadband presence from an earlier time to the present across the states. A series of ‘task’ are presented to illustrate how to download the data, download the state geometries, join data with the map and finally present the results. The companion package RspatialPkg is a ggplot2 based R package that creates and maps the visualizations. As the investigator gains more knowledge of his/her subject, not all of the steps outlined here will be required. We are starting from ground zero with this demonstration where we know little about the availability of Census Bureau data or how to access it and put it together. RcensusPkg is available from GitHub at deandevl. There are many additional demonstrations on the package at the repository. In the demo folder each function is demonstrated for downloading data and sf geometries along with R script tutorials. Also R_Analysis_of_US_Census_Data contains R script that uses RcensusPkg to follow Kyle Walker’s book Analyzing US Census Data: Methods, Maps, and Models in R.

1.2 Formulate a goal

Formulate a goal that characterizes the types of variables and maps needed.

Our goal is to get annual survey percentages on computer ownership and presence of broadband in households across the states. We want a recent year like 2021 and an earlier year such as 2013.

1.3 Find a dataset name acronym

Do a general search at the US Census Bureau’s site to identify possible dataset sources.

An excellent place to start are the descriptions of all the datasets available from the API at Available APIs. In addition, there is the Census Bureau’s Explorer where the Tables are presented. Tables or datasets are really groups of survey variables that could possibly relate to our goal. We are wanting to find the dataset acronym that has computer related variables. We see a Table named “DP02 Selected Social Characteristics in the United States”. When we click the link for the DP02 Table we get a listing which at the very end has the sub-category “COMPUTERS AND INTERNET USE”. Two variables under the category have numbers for households “With a computer” and “With a broadband internet subscription”. We also see back toward the top of the page the name of the dataset as “ACS 1-Year Estimates Data Profiles”. It appears that this dataset is what we are looking for.

Determine the acronym for the dataset “ACS 1-Year Estimates Data Profiles”.

Use RcensusPkg::get_dataset_names() and the “title” column of the resultant data.table containing “1-year” as a case non-sensitive filter string across all years and datasets.

acs_dataset_names_dt <- RcensusPkg::get_dataset_names(
  filter_title_str = "1-year"
)

Among the 109 “1-Year” surveys we find that “acs/acs1/profile” has extensive years (2005 to 2021) and contains the key word “profile”. The one exception is 2020 which is missing because of the Bureau’s Decennial or 10 year survey for that year.

1.4 Find the computer related variable names

We have a dataset acronym name and some dates. Find the availability of variable names that relate to computer and broadband presence in households.

We learned from the above discussion that table “DP02” has a sub-category “COMPUTERS AND INTERNET USE”. We can use this information to search the “acs/acs1/profile” dataset using RcensusPkg::get_variable_names(). The function requires a dataset name, a vintage and a search string. We’ll start with vintage 2021 and the “label” column of the resultant data.table containing “computers” as a case non-sensitive filter string.

DP02_2021_computers_dt <- RcensusPkg::get_variable_names(

1.5 Find “required” variables

Make sure that there are no required variables when requesting data.

Whenever requesting variable names, you should review the “required” column for any variables that should be included in your request. If left out you will receive an error from the API. Using the 2021 vintage as an example we can quickly check for required variables using simple data.table filtering. The value for “required” should be either NA or “default displayed” for no requirements.

DP02_2021_variables_dt <- RcensusPkg::get_variable_names(
  dataset = "acs/acs1/profile",
  vintage = 2021
)
required_variables_dt <- DP02_2021_variables_dt[!is.na(required), ]

1.7 Request the computer related data

We have all the parameters now for submitting a request from the “acs/acs1/profile” dataset for 2013 and 2021.

Get the data for 2013:

acs1_computers_data_2013_dt <- RcensusPkg::get_vintage_data(
  dataset = "acs/acs1/profile",
  vintage = 2013,
  vars = c("DP02_0150E", "DP02_0151PE", "DP02_0152PE"),
  region = "state:*"
)

Rename columns; select columns; convert values to numeric; order the rows by “State”:

acs1_computers_data_2013_dt <- acs1_computers_data_2013_dt %>% 
  data.table::setnames(
    old = c("NAME", "DP02_0150E", "DP02_0151PE", "DP02_0152PE"),
    new = c("State", "Total", "ComputerPresent", "BroadbandPresent")) %>% 
  .[, .(GEOID, State, Total, ComputerPresent, BroadbandPresent)] %>% 
  .[, `:=`(Total = as.numeric(Total), ComputerPresent = as.numeric(ComputerPresent),
           BroadbandPresent = as.numeric(BroadbandPresent))] %>% 
  .[order(State)]

1.8 Download the boundary geometries for US states and join with the 2013 and 2021 computer present data

Our goal here is to join the boundary geometries by state with the above survey data which is also by state. We can get state boundary data by using RcensusPkg::tiger_states_sf().

Join the 2013 and 2021 computer present data with state geometries.

Some preliminaries: Note that an existing local folder must be specified to receive all downloaded Census Tiger related shapefiles. Also when requesting geometries we can define an “express” parameter to limit the request with a logical expression.

output_dir <- file.path(here(), "demo", "shapefiles")
express <- expression(State != "Alaska" & State != "Hawaii")

A data.frame with data and state boundaries for 2013 joined:

state_boundaries_2013_sf <- RcensusPkg::tiger_states_sf(
  output_dir = output_dir,
  general = TRUE,
  express = express,
  datafile = acs1_computers_data_2013_dt,
  datafile_key = "GEOID",
  sf_key = "STATEFP",
  check_na = TRUE,
  sf_info = FALSE
)

A data.frame with data and state boundaries for 2021 joined:

state_boundaries_2021_sf <- RcensusPkg::tiger_states_sf(
